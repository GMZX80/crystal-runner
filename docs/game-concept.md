# Game Concept

Game title: Crystal Runner

Target platform: Browser first, GitHub Pages for playtesting, later Android/iOS packaging preparation through Capacitor.

Target player: Casual arcade players using desktop or mobile browsers.

Player fantasy: A pirate ship survives on a floating sea path by reacting quickly, collecting bright gold crests, and slipping past cannon hazards.

Core verb: Switch lanes.

Core loop: Read the lane pattern -> move left or right -> avoid cannon hazards -> collect gold crests -> score rises -> speed increases -> fail and retry for a better run.

30-second experience: Start immediately, move between three lanes, collect one crest, dodge one cannon hazard, and understand that collision ends the run.

First 5-minute experience: Learn lane switching, recognise safe lanes, chase crest pickups, and retry quickly after failure.

Main source of challenge: Increasing speed and lane pattern reading.

Feedback system: HUD score/crests, collection sound, lane movement sound, collision sound, game-over panel, and debug performance text.

Reward system: Score, crest count, and local best score.

Failure/retry model: Hitting an obstacle ends the run and presents an immediate restart button.

Accessibility concerns: Touch target size, readable text, colour-independent clarity for cannon hazards/crests, reduced motion option not yet implemented.

Ethical engagement concerns: No monetisation, no tracking, no timers outside core arcade pressure, no dark-pattern retention.
