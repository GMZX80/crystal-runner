# Playtesting

Playable URL: pending first GitHub Pages deployment

Test objective: Confirm that first-time players understand lane switching, crystal collection, obstacle avoidance, failure, and retry within three minutes.

Suggested play duration: 3-5 minutes or three complete runs.

Build/commit tested: pending first deployment

## Questions For Playtesters

- Did you understand what to do?
- Did you know why you failed?
- Did you want to try again?
- What felt good?
- What felt confusing?
- What felt unfair?
- What felt boring?
- What would you change first?
- Did keyboard or touch controls work for you?
- Did performance or loading cause problems?
- Did anything create an accessibility barrier?

## Reporting Bugs

Use the GitHub bug report issue template. Include build URL, commit/version, device, browser, operating system, steps to reproduce, expected behaviour, actual behaviour, screenshots/video, console errors, and severity.

## Reporting Design Feedback

Use the playtest feedback issue template. Prioritise feedback that affects clarity, fairness, desire to retry, control feel, readability, accessibility, or performance.
